# GOVERNANCE

AIUZ repository governance bridge.

## Repository role

`AIUZ-` is a practice-trunk repository.

Its role is to host implementation-oriented, public-facing material related to:

- ethical educational technology;
- AIUZ platform logic;
- multilingual learning architecture;
- practical documentation that belongs in a public working repository.

It is not the place for:

- full theoretical archives;
- raw shadow exports;
- uncontrolled backups;
- duplicated timestamped dump material.

## Authority order

When governance sources differ, use this order:

1. child safety and human dignity
2. Terra Universal Convention
3. repository security and conduct rules
4. repository publication boundary
5. implementation convenience

If convenience conflicts with protection, protection wins.

## Decision classes

- Minor editorial changes: maintainer review
- Structural documentation changes: maintainer review plus repository context check
- Governance changes: must remain compatible with `terra-legal`
- Child safety issues: immediate override and accelerated response

## Source integrity rule

AI and automation must not invent repository reality.

They may:

- summarize existing materials;
- reorganize clearly duplicated public documentation;
- propose new structures when marked as proposals.

They may not:

- fabricate project history;
- silently merge theory and practice trunks;
- import shadow residues as if they were living truth.

## Cross-repository relation

This repository inherits its legal and governance donor layer from the broader Terra governance trunk, especially:

- `terra-legal`
- organization community guidance
- Terra Universal Convention line

## Emergency rule

Any issue affecting children, vulnerable users, personal data, or manipulative design bypasses normal cadence and receives immediate maintainer attention.
