# PUBLICATION RULE

Publication boundary for `AIUZ-`.

## Public by default only when curated

This repository is public-facing, but that does not mean every file is publication-ready.

Publication must remain curated.

## Safe public surface

Generally publishable:

- `README.md`
- multilingual `README` files
- repository governance files
- public documentation intended for humans
- public web entry files such as `index.html`, `style.css`, `script.js`
- explicitly curated public markdown and public assets

## Not automatically publishable

Do not publish blindly:

- raw backups
- session residues
- timestamped duplicates
- shadow exports
- private notes
- internal audit fragments
- experimental drafts without context
- personal or third-party sensitive data
- secrets, tokens, credentials, or machine state

## Fractal rule

If a file belongs more properly to:

- theory,
- legal/governance,
- corpora,
- archive/shadow,

then it should not be forced into the `AIUZ-` publication surface just because the repository is public.

## Before publish

Check:

1. Is it readable by humans?
2. Is it ethically safe?
3. Is it the right trunk?
4. Is it free from shadow duplication?
5. Does it strengthen, rather than blur, the repository role?

If any answer is no, do not publish blindly.
